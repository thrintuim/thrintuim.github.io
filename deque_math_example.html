aria-label as fall back for math content

The following is modified from an example in Deque University for how math should be markedup. The goal is to show that this method is actively harmful to the user experience.

Example 1: Math element with parent that has aria-label

Expectation from Deque: Screen readers that announce MathML will skip the @aria-label and process the MathML.

Actual result: Screen readers that announce MathML read the @aria-label and ignore the inner MathML.

1 2 + 1 2 = 1

Example 2: raw mathml

Expectation: Screen readers that can process MathML will announce "one half plus one half equals 1" or some approximation. Screen readers that cannot process MathMl will announce nothing.

1 2 + 1 2 = 1
