iOS VoiceOver parenthesis announcement in math testing

The following is to demonstrate how iOS VoiceOver does and does not handle parentheses in text and MathML

Parens in text (this is a parenthetical statement).

mfenced markup: 1+1

mrow mo markup: (1+1)

Set notation mrow mo markup: {x|−3<x≤9}

with different mrow markup: {x|−3<x≤9}

same mrow markup but replace curly brackets with parens: (x|−3<x≤9)

with mfenced markup: x|−3<x≤9

with mfenced markup and | separator: x−3<x≤9

function notation mfenced: fx

function notation mfenced and function application: f⁡x

function notation mrow mo: f(x)

function notation mrow mo function application: f⁡(x)

mfenced with multiple arguments and default separators. Expectation is "left paren one comma two comma three comma four right paren". 1234

mfenced with specified separators. Expectation is "left paren one semi-colon two comma three slash four right paren". 1234

mfenced with open close with |: x−3<x≤9

mfenced with open close with ⟦ and ⟧: x−3<x≤9

mfenced with open close with 〈 and 〉: x−3<x≤9
